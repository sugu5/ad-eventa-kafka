  ✓ events_sent_total             Total messages sent (counter)
  ✓ events_failed_total           Total delivery failures (counter)

  ✓ producer_queue_size           Internal queue backlog (gauge)
  ✓ producer_offset               Offset per partition

CONSUMER METRICS (Kafka Exporter):
  ✓ kafka_partition_current_offset    Latest broker offset
  ✓ kafka_consumergroup_current_offset Consumer group position
  ✓ kafka_consumergroup_lag           Lag per partition
  ✓ kafka_topic_partitions            Partitions per topic
  ✓ kafka_broker_info                 Broker metadata

CALCULATED METRICS (Grafana):
  ✓ Latency Percentiles (p50, p95, p99)
  ✓ Events Rate (events/sec)
  ✓ Failed Event Rate
  ✓ Total Consumer Lag per Group

Before starting producer/consumer, verify:

  ☐ Services are running:
    docker-compose ps kafka-exporter prometheus grafana
    (All should show "running" or "healthy")

  ☐ Prometheus is UP:
    http://localhost:9090/targets
    (kafka-exporter:9308 and host.docker.internal:8001 should be UP)

  ☐ Grafana dashboard loads:
    http://localhost:3000
    (Should see "Kafka Streaming Dashboard" in list)

  ☐ Prometheus datasource is healthy:
    Grafana → Settings (gear) → Data Sources → Prometheus
    (Should show "datasource is working" with green checkmark)

If any check fails, see BEAUTIFUL_DASHBOARD.md troubleshooting section.
